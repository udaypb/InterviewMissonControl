# Interview Mission Control — Operating Manual (Final v3)

---

## 1. System Intent (Non-Negotiable)

This system exists to do one thing:

> **Continuously convert ambiguity into clear, high-impact actions during your interview process.**

It is NOT:
- a note-taking system
- a productivity tracker
- a place to “feel organized”

It IS:
- a **decision engine**
- a **priority generator**
- an **execution system**

---

## 2. What Problem This Solves

Without this system:
- you prepare randomly
- you chase low-impact work
- you lose time switching contexts
- you don’t align prep with interviews

With this system:
- every action ties to an outcome
- priorities are always visible
- prep is aligned with actual interviews
- thinking overhead is minimized

---

## 3. Inputs (How the System Receives Information)

This system is designed for **imperfect, real-world inputs**.

---

### 3.1 User Input (Primary Input)

Your input will often be:
- vague
- incomplete
- emotional
- unstructured

Examples:
- “I feel weak in DP”
- “Expedia is coming up”
- “I don’t feel confident”
- “What should I do today?”
- “TikTok might be important”

**Rule:**
> The system must interpret, not require precision.

---

### 3.2 Google Calendar (Ground Truth)

Calendar provides:
- interview timing
- urgency windows
- constraints

**Rule:**
> Calendar overrides everything.

---

### 3.3 Context Docs (Memory Layer)

Provide:
- strategy
- long-term goals
- past learnings
- patterns in behavior

---

### 3.4 Existing Sheet State

Provides:
- current decisions
- existing priorities
- current execution plan

---

## 4. System Architecture

---

### Layer 1 — Context (Docs)

Purpose:
- store thinking, strategy, reflections

Nature:
- unstructured
- evolving
- high-signal

---

### Layer 2 — Execution (Sheet) ← CORE

Purpose:
- convert interpretation into decisions

Nature:
- structured
- minimal
- action-focused

---

### Layer 3 — Dashboard

Purpose:
- display decisions clearly

Nature:
- read-only
- derived from sheet

---

## 5. Core Rule

Docs = thinking  
Sheet = decisions  
Dashboard = display  

Never violate this.

---

## 6. The Role of the Sheet (Critical)

The sheet is NOT storage.

> **The sheet is the system’s interpretation of reality.**

It is:
- where ambiguity is resolved
- where priorities are enforced
- where execution becomes concrete
